# Data Products (Gumroad + RapidAPI)

The agent scrapes publicly available datasets (job boards, pricing data, public repos), packages them into structured CSVs and JSONs, and sells them on Gumroad and RapidAPI.

Monthly subscriptions for maintained feeds; one-time purchases for static datasets.

## How It Works

**Data Collection:**
1. Agent identifies valuable public datasets
2. Scrapes or retrieves data from public sources
3. Cleans and structures the data (CSV, JSON, etc.)
4. Validates for quality and accuracy

**Sales Model:**
- **Gumroad**: One-time purchase of static datasets
- **RapidAPI**: API endpoint (monthly subscription model)

**Maintenance:**
- For subscriptions: Keep data updated and fresh
- For one-time: Occasional updates for major changes

## Types of Data Products

Agents can create products like:
- **Job market data**: Aggregated postings from job boards
- **Pricing data**: Historical prices, market trends
- **Repository data**: Code quality metrics, popular libraries
- **Market data**: Cryptocurrency, commodities, stocks
- **Social data**: Trending topics, sentiment
- **Web data**: Indexed content, metadata

## Why It's Slowest

1. Takes time to identify valuable datasets
2. Scraping and cleaning takes effort
3. Building customer base is slow
4. Requires marketing to find buyers
5. Initial revenue is near-zero while building portfolio
